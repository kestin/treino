Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Para a funo performance_metric na clula de cdigo abaixo, voc ir precisar implementar o seguinte:

  • Utilizar o r2_score do sklearn.metrics para executar um clculo de desempenho entre y_true e y_predict.
  • Atribuir a pontuao do desempenho para a varivel score.
In [18]:
from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calcular e retornar a pontuao de desempenho entre 
        valores reais e estimados baseado na mtrica escolhida. """
    score = r2_score(y_true, y_predict)
    # Devolver a pontuao
    return score

Resposta: Nunca devemos usar nosso conjunto de teste para treinar o modelo. Entao o ideal termos um conjunto de validao para tomada de deciso em relao ao modelo, com o valor dos parametros por exemplo. Assim nossos dados ficaro divididos em 3 conjuntos: Treinamento, Validao e Teste A tcnica de validao-cruzada k-fold um mtodo para reciclarmos nossos dados. O objetivo estimar o quo preciso o modelo ser quando novos dados entrarem. Funciona da seguinte forma:

Os dados so quebrados em K subconjuntos mutualmente exclusivos. Treinamos nossos modelos K vezes, cada vez usando um pedao diferente como conjunto de testes e o restante como conjunto de treinamento. Tiramos uma mdia dos resultados para escolher o modelo final

Essa tcnica se ajusta muito bem a utilizao da busca em matrix sem um conjunto de validao cruzada, pois ao dividir os dados em conjunto de treinamento e conjunto de testes apenas uma vez o tamanho dos conjuntos fica limitado e ao usar um nico conjunto de validao pode ocorrer overfiting.

Assim o grande benefcio de usar a valizao-cruzada k-fold na busca em matriz que todas as observaes so usadas para treinamento e validao, e cada observao usada para validao uma nica vez. Isso nos d a segurana de usar todo o dataset para treinamento e validao sem correr o risco de superestimar o desempenho do conjunto de validao nico que foi selecionado na otmizao do modelo.
